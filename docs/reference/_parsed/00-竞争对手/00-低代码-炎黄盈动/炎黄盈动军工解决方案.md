2、 应用容器技术 
AWS PaaS 在底层技术架构上采用PaaS 应用容器技术，能够对多个流程应
用的服务进行集中管理，采用应用容器的优势之一是能够避免单一应用的部署或
局部问题殃及整个平台，可以在应用故障解除之前，暂停有问题的应用。在AWS 
PAAS 中，该功能方案被称为AMC（Application Management Control）。 
作为一个企业级的统一流程服务平台，将若干BPM 应用通过AMC 进行管
理的优势包括： 
（1）独立性：每个流程应用的资源被独立描述、独立加载、独立控制、独
立日志，当应用出现故障时，更容易排查。 
（2）易管理：每个流程应用都可以独立的控制启动、暂停、重启等操作，
使局部问题不会影响全局。 
（3）易分发：从开发环境、测试环境到生产环境，可以被自动化成打包文
件，安装或升级到目标BPM 平台。 
（4）易扩展：应用发生变化时，只需要升级差异的应用即可，过程由应用
容器自动完成。 
（5）易组合：通过AMC 提供的应用依赖关系检查，可以可视化的给出应
用与应用间的依赖拓扑图，为运维和开发者提供真实的架构信息。 
（6）易升级：可以控制每一次应用包的升级过程和历史，所有元数据版本
操作都自动保留可追溯的历史存档。 
（7）易部署：含有jar 程序的应用不必重启BPM 服务即可热加载，应用容
器为每个应用分配独立的Class Loader，防止冲突的jar 版本干扰其他开发者的程
序。

3、 三员管理 
应用三员管理目的是实现基于AWS BPM 流程解决方案的三权隔离、相互制
约的监督关系，避免由于管理员权限过于集中带来的安全风险。

（1）三员 
参照安全保密相关规定，三员身份被设计为：系统管理员(sysadmin)、安全
保密员(secadmin)和安全审计员(auditor)，三者相互制约。

应用系统的建模配置、运行管理及非安全参数调整等工作，实施和配置业务
流程系统，查看与应用系统异常相关的日志等。系统管理员可能是一个账户，也
可以是相互隔离的多个二级管理员。

（2）三员管理特点 
➢ 严禁系统中存在具有高权限的超级管理员账号，超级管理员仅可作为系统
安装、初始过程的临时账户。 
➢ 审计管理员自己的登录、退出、日志查看等日志信息只能有安全管理员查
看，审计员、管理员自己不能查看。 
➢ 日志信息不能被任何管理员删除。
