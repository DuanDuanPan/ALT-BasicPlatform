2.2　工业APP 封装痛点
当前市面上已经有一些工业APP 封装软件，但这些
软件封装过程操作较为繁琐、且功能相对不是很健全，
用户使用体验感较差。不足的地方主要体现在如下几个
方面：
1）应用场景局限：低代码无编程封装工具的应用场
景比较局限，只能支持特定类型的仿真模型，无法满足
多样化的仿真需求。
2）缺乏灵活性：低代码无编程封装工具的灵活性不
够高，无法满足复杂仿真场景的需求，从而导致仿真模
型的精度和效果不佳。
3）技术门槛高：低代码无编程封装工具需要具备一
定的技术背景和操作经验，对于非专业人士来说，学习
和使用成本较高。
针对上述难点，基于仿真程序/ 仿真模型自适应封装
技术、参数自动化识别与驱动技术、拖拽式可视化UI 界
面搭建技术、参数数据与界面控件动态绑定关联技术，
开发可视化组件封装系统AutoWrap，实现模型参数显性
化、基于参数的标准化封装和自动执行的可视化APP。

模型标准化封装过程主要分为四个子过程。
1）参数识别过程：从定义的输入模型中识别参数，
形成参数集合用于后续驱动。
2）程序调用过程：通过选择或者插件自动化识别到
仿真程序的启动路径和带参数启动方式，用于后续程序
调用。
3）参数界面绑定过程：通过拖拽式方式实现参数集
合中的参数与UI 界面中定义的控件实现绑定，方便后续
可视化设置和查看。
4）模型封装发布过程：通过设定的参数和绑定的UI
控件，基于标准化封装技术实现模型封装的自动化发布。
模型动态调用过程主要分为四个子过程。
1）封装模型实例化过程：框架通过后台解析标准化
封装模型，实现UI 界面的自动化加载。
2）界面参数输入过程：根据当前的模型需求在弹出
的UI 界面中设置对应的输入参数值。
3）模型运行过程：框架根据输入参数自动修改输入
模型，并自动化调用仿真程序生成输出数据，解析到当
前输出参数中。
4）输出参数可视化过程：基于识别到的输出参数，
根据UI 界面的数据绑定规则实现当前输出参数的动态可
视化展示。
3.2　系统功能组成
面向工业设计仿真模型封装构建的一体化集成系统，
需要实现常见工业模型的识别封装，数据的动态可视化
交互显示，APP 页面的控件逻辑配置，并最终完成APP
运行包的测试与标准化发布。系统主要包含工具集成、
封装器、界面设计和逻辑设计等功能，如图4 所示。
图4　可视化封装系统功能架构

（1）一体化主环境　一体化封装主环境是实现工业
设计仿真APP 可视化封装的基础工作台，通过主环境构

实现在同一环境下模型参数识别、界面搭建、数据关联、

一体化主环境是用户封装的基础工作台，在工作台
中，用户选择封装器，定义调用程序、模型及脚本，可
视化脚本参数，拖拽式关联参数与界面UI，设置APP 运

（2）模型封装器　模型封装器通过提供不同的封装
器插件实现异构模型的动态参数识别和模型驱动接口。
通过Interface 接口实现不同封装器的动态接入来扩展模
型识别类型。
通用封装器：针对输入、输出文本/ 脚本的多类型参
数可视化识别与修改，通过带输入/ 输出启动的方式实现
工具/ 模型的动态封装，大多数封装都可以基于GSP 通
用封装器完成。
CAD 封装器：通过识别到参数化模型的几何参数、
装配参数、材料参数和属性参数等，进行自动化驱动，
动态更新模型并统计体积、质量等模型信息。
公式封装器：基于所见即所得的公式编辑器场景编
辑公式，利用Latex 参数识别与驱动技术，实现公式规格
化校验、参数自动化识别、后台自动转换计算。
Excel 报表封装器：通过对Excel 单元格的自定义识
别，实现基于Cell 的动态输入输出参数定义，通过修改
输入、更新Excel 模型文件、抽取输出结果。
Word 文档封装器：通过自动识别Word 文档中的预
定义“书签”，完成参数定义，通过设置对应的单值、表格、
图标和文字段实现Word 文档的自动化更新。
程序代码封装器：针对输入MATLAB 脚本.m 文件，
可视化实现参数识别与修改，自动驱动MATLAB 进行运
行计算得到结果。

信号处理：直接关联信号与槽，这也是最基本的信
号处理方式。
动态调用：可以调用用户界面上其他Widget 的方法。
支持AutoWrap 参数，以及字面值常量作为函数的输入。
如果函数有返回值，那么还可以将返回值放回参数，或
者写回到其他Widget 的属性中。
EXE 程序调用：可以调用外部的可执行程序，或者
bat 脚本。并支持AutoWrap 的参数的值，作为被调用程
序的命令行参数进行传参。
扩展库调用：支持调用扩展的DLL 文件。扩展库需
要实现SDK 的ILogicLibrary 接口，并将生成的DLL 放
到AutoWrap 的plugins 目录LogicExtension 子目录下。
