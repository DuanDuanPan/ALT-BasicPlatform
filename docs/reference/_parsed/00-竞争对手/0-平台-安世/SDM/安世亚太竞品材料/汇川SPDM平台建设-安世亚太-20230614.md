仿真知识管理业务痛点


仿真知识管理业务痛点

仿真数据分散在不同工程师电脑上，获得数据需要找对应的工程师，工程师也经常找不到放哪里了。 同一零部件做的不同分析（结构/多体等），保存在不同的工程师处，查找完整的数据需要找不同的人员。 仿真数据无法清晰的与产品结构进行关联，类似产品的仿真数据也无法参考或调用，仿真数据搜索查阅很困难。 多方案分析过程产生的数据无法按方案进行组织和管理。 仿真模型必须由仿真工具打开才可以预览，不能进行快速轻量化的预览。 | 基于共享文件夹分析数据的方式，由于多个人对共享文件夹都有较高的权限，文件被修改或删除了，但却不知道是谁做的。 某个仿真分析人员离职后，他曾经负责的产品仿真数据可能一起流失，或者其他人员难以找到。 仿真数据共享交换困难，不同专业之间存在大量的“数据孤岛”。



**Notes:**
项目业务痛点分为六大方面，分别集中在仿真数据管理、仿真工具链集成、设计仿真协同、数据分析决策、计算资源管理和仿真知识管理方面，每个业务痛点会在企业的研发设计中体现为很多具体的问题。这两个方面在上述六大场景的介绍里也进行了说明，不再赘述。

仿真知识管理业务痛点

产品仿真过程中使用的商用仿真工具较多，在仿真过程主要是各种软件的单点应用，工具选择随意性大，给模型统一化集成造成困难。 产品的仿真分析问题涉及静强度、疲劳、电磁、散热、动力学等非常复杂的问题，当前没有专职的仿真人员，大部分设计人员很难达到深层次应用水准。 仿真分析的工况相对比较标准，很多分析流程比较类似，但设计过程还是必须对每个产品进行重复性工作，如建模、划分网格、边界设置、求解及结果后处理等，人力成本资源无形浪费。 | 单点工具之间存在大量的模型/数据传递、接口数据转换的工作，没有统一规范化，且效率低下。 当前需要对仿真知识/模型进行复用，但当前公司又不具备既懂业务，又懂模型封装开发的设计人员。

仿真文件之间无法进行关联，不便于查找确认计算结果与计算条件的对应关系。 仿真数据无法进行二次分析，无法找到影响性能的各因素之间的关系。 多方案仿真结果无法进行对比分析，无法快速形成可参考的仿真对比建议。 没有打通基于仿真数据优化产品设计的技术过程。



**Notes:**
项目业务痛点分为六大方面，分别集中在仿真数据管理、仿真工具链集成、设计仿真协同、数据分析决策、计算资源管理和仿真知识管理方面，每个业务痛点会在企业的研发设计中体现为很多具体的问题。这两个方面在上述六大场景的介绍里也进行了说明，不再赘述。
