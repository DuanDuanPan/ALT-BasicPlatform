 研制背景

    大力推进研究院管理业务信息化建设，全面带动管理质量和效率的提升，促进科研水
平和创新能力的提高，已成为研究院落实科技强军目标的重要战略举措之一，也是研究
院实现“成为国内、军内顶尖智库”这一战略发展目标的迫切需要和必然选择。
    目前，研究院信息化发展存在的问题主要体现在：
    1. 应用系统难以满足业务需求；
    2. 信息资源分散，没有实现全面互联互通，难以共享和有效利用；
    3. 缺乏有效的信息化管控体系。
    研究院信息化面临的形势主要体现在：
    1. 新军事革命发展对信息化的需求；
    2. 实现研究院发展战略对信息化的要求；
    3. 信息技术发展为信息化带来的机遇。
    为解决制约研究院信息化科学发展的全局性、系统性问题，并通过信息化手段促进研
究院发展战略落地，迫切需要开展数字化业务管理系统建设，统筹研究院信息化工作，
以指导和规范研究院信息化建设管理，确保信息化建设管理的思路统一性、目标合理性
、工作规范性等。
    数字化业务管理系统建设目标：以研究院发展战略为牵引，管理业务为主线，信息数
据为核心，建立信息互联、敏捷高效、智能决策、架构灵活，集业务协同管理和综合服
务于一体的应用系统集群，实现业务全覆盖、信息全打通、系统全集成、架构全统一，
推动信息资源共享和“智量”增长，提升协同效率和服务质量，全面促进管理水平进一步
提高，为实现研究院战略目标提供有力的信息化支撑。

 研制过程

    2016年9月项目正式启动，中船重工奥蓝托无锡软件技术有限公司组织成立科研业务
管理系统项目开发团队，对用户业务需求进行收集和熟悉，在研究院相关人员的积极配
合下，进行了需求分析、软件设计、功能开发、系统测试、系统试运行等活动，顺利完
成了项目开发的全过程。
    系统从开始到结束，共经历了需求分析与方案设计、详细设计、开发、测试运行、验
收总结等几个阶段。
    1. 需求分析与方案设计
    需求分析与方案设计的执行时间是从2016年9月—2016年11月。2016年9月，奥蓝托成
立项目组，前往研究院进行需求调研，就项目详细需求及应用场景进行了细致讨论。20
16年11月，在研究院召开了需求方案评审会，顺利通过需求方案评审。2016年11月，奥
蓝托完成方案设计，包括总体设计、技术规格书、需求规格说明书等。
    2. 详细设计
    详细设计的执行时间是从2016年11月—2017年3月。2017年3月，奥蓝托在方案设计基
础上，开始进行详细设计，包括各分系统每一块内容的功能描述、流程设计、界面设计
、方法设计、服务设计、数据对象状态设计、数据库设计等，同时进行关键技术研究。
2017年3月，在研究院召开详细设计评审会，顺利通过详细设计评审。
    3. 开发
    功能开发的执行时间是从2016年11月—2017年7月。2016年11月开始，奥蓝托的实施、
研发、测试、美工等人员通力合作，根据客户需求进行定制开发，不断对软件进行内部
测试和改进。开发期间，定期给研究院汇报工作进展。
    4. 部署
    从2017年7月开始，奥蓝托陆续发布并部署了多个试用版本，覆盖了所有功能模块。

    5. 测试
    系统分两个阶段：第一阶段，依据《软件测试用例》对系统进行了内部测试，并及时解
决了测试过程中发现的问题，形成了软件试用版本；第二阶段，软件版本部署后，进行
现场测试，及时处理试用过程中出现的软件问题和用户的修改意见，并发布新的版本。

    6. 用户培训
    奥蓝托对系统相关用户进行了培训，主要分为以下几类：
     ✓ 基础培训
    培训对象：系统使用人员；
    培训内容：系统工作原理和使用方法。
     ✓ 系统管理员（IT运维）培训
    培训对象：系统管理员，IT维护人员；
    培训内容：系统安装，应用服务器及数据库服务器的日常维护。
    7. 上线
    经过与项目负责人和使用部门进行多次讨论、修改之后，系统于2017年12月在正式服
务器中上线运行。在上线运行后，继续搜集使用人员提出的问题和改进意见，对系统进
一步完善，以达到他们的使用要求。
