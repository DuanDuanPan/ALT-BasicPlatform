部署环境搭建

Linux下运行环境的搭建，使用docker容器实现，所有运行环境，均安装在docker容器中，使用时，直接导入镜像，运行docker容器实例使用即可，如需手动安装，请参考“https://192.168.8.8:444/svn/SUN_运维/平台/0 部署环境/”下各软件的安装说明；

准备工作

在服务器中安装Docker；

关闭selinux：

方法一：

“vi /etc/selinux/config”；

”#SELINUX=enforcing“， #注释掉；

”#SELINUXTYPE=targeted“， #注释掉；

”SELINUX=disabled“，设置selinux不可用；

”:wq!“，保存退出；

”setenforce 0“，关闭selinux；

方法二：

”sed -i 's/SELINUX=enforcing/SELINUX=disabled/g' /etc/selinux/config“，修改设置；

”setenforce 0“，关闭selinux；

数据库文件服务器

在容器启动后，关闭防火墙（systemctl disable iptables 、service iptables stop），否则Tracker与Storage之间无法通信；

MySQL

主服务器

将dataBak文件夹，复制到/root/workTest目录下；

”cat /root/workTest/mysql-5.6.tar | docker import - mysql-5.6:cssrc“，导入镜像；

”docker run --name mysql-5.6 --restart=always --net="host" -i -t -v /root/workTest/dataBak:/root/workTest/dataBak mysql-5.6:cssrc /etc/rc.d/rc.local “，运行容器；

FastDFS

主服务器

建立FastDFS文件挂载目录：



” cat /root/fastdfs-5.08-tracker.tar | docker import - fastdfs-5.08-tracker:cssrc“，导入镜像；

“docker run --name fastdfs-tracker --restart=always --net="host" -i -t -v /root/workTest/fastDFS/fastdfs_tracker/:/root/workTest/fastDFS/fastdfs_tracker/ fastdfs-5.08-tracker:cssrc /etc/rc.d/rc.local”，运行tracker容器，由于FastDFS无法返回外网地址，只能返回容器内部内网地址（如：127.17.0.4），所以，容器实例化时”--net="host"“选项为必须添加的，使用宿主机的网络；

 “cat /root/fastdfs-5.08-storage.tar | docker import - fastdfs-5.08-storage:cssrc”、“docker run --name fastdfs-storage --restart=always --net="host" -i -t -v /root/workTest/fastDFS/fastdfs_group1/data:/root/workTest/fastDFS/fastdfs_group1/fastdfs_storage_data -v /root/workTest/fastDFS/fastdfs_group2/data:/root/workTest/fastDFS/fastdfs_group2/fastdfs_storage_data -v /root/workTest/fastDFS/fastdfs_group1/logs:/root/workTest/fastDFS/fastdfs_group1/fastdfs_storage/logs -v /root/workTest/fastDFS/fastdfs_group2/logs:/root/workTest/fastDFS/fastdfs_group2/fastdfs_storage/logs fastdfs-5.08-storage:cssrc /etc/rc.d/rc.local”，运行storage容器，由于FastDFS无法返回外网地址，只能返回容器内部内网地址（如：127.17.0.4），所以，容器实例化时”--net="host"“选项为必须添加的，使用宿主机的网络；

进入容器，“vi /etc/fdfs/storage_group1.conf”, “vi /etc/fdfs/storage_group2.conf”修改Storage配置，设置Tracker地址；



“docker stop ******”、“docker start ******”， 重启Storage容器；

从服务器

“cat /root/fastdfs-5.08-storage.tar | docker import - fastdfs-5.08-storage:cssrc”，导入storage镜像；

“docker run --name fastdfs-storage --restart=always --net="host" -i -t -v /root/workTest/fastDFS/fastdfs_group1/data:/root/workTest/fastDFS/fastdfs_group1/fastdfs_storage_data -v /root/workTest/fastDFS/fastdfs_group2/data:/root/workTest/fastDFS/fastdfs_group2/fastdfs_storage_data -v /root/workTest/fastDFS/fastdfs_group1/logs:/root/workTest/fastDFS/fastdfs_group1/fastdfs_storage/logs -v /root/workTest/fastDFS/fastdfs_group2/logs:/root/workTest/fastDFS/fastdfs_group2/fastdfs_storage/logs fastdfs-5.08-storage:cssrc /etc/rc.d/rc.local”，运行一个storage容器，由于FastDFS无法返回外网地址，只能返回容器内部内网地址（如：127.17.0.4），所以，容器实例化时”--net="host"“选项为必须添加的，使用宿主机的网络；

进入容器，“vi /etc/fdfs/storage_group1.conf”, “vi /etc/fdfs/storage_group2.conf”修改Storage配置，设置Tracker地址；



“docker stop ******”、“docker start ******”， 重启Storage容器；

测试

进入Storage容器，”/usr/bin/fdfs_monitor /etc/fdfs/storage_group1.conf“，查看当前tracker服务器与storage服务器连接状况；

“cp /etc/fdfs/client.conf.sample /etc/fdfs/client.conf”；

“vi /etc/fdfs/client.conf”,设置客户端的配置文件，做以下修改：

base_path=/root/workTest/fastDFS/fastdfs_tracker 	#文件日志存放位置，暂时tracker服务器文件路径
