SUN.BPM3.X企业应用统一开发平台是一款基于微服务架构的业务流程快速开发平台。


**Notes:**
1.低代码平台：这类平台实际是面向开发和技术人员使用，并不是说完全无代码化，而是真正的叫低代码，实现低代码核心还是共性技术组件能力下沉和可复用。
   如何评价一个好的低代码平台：这类低代码平台不要去管具体的业务流程，业务功能和业务逻辑的实现，只要做好共性技术组件，做好开发框架和底层技术支撑，就是一个很好的平台。
2.零代码开发平台：这类平台更加偏向于面向业务人员，或者更多是不直接面对底层代码的零代码开发平台，希望做到的是，应用可以通过搭建积木的方式进行快速配置来实现。
3.但是最近通过复盘和一些产品试用，整体感觉对于中小企业很多仅仅是实现表单电子化和流程化的场景（单据本身也不和其他单据发生强耦合和关联关系。属于典型的电子表单+流程的需求场景。），完全可以做到类似上面的零代码和快速配置。

优势

技术选型上，更多考虑行业的适用性
架构设计上，更多考虑组件的复杂性
实施开发上，基于原生语言，包容万象

特性


同属于零/低代码开发平台，表单、流程功能成熟
技术架构和实施方法接近
对复杂业务都需要定制开发

共性

技术选型上，更多考虑技术的先进性
架构设计上，更多考虑组件全而多
实施开发上，自成体系，容易被绑架

特性

体系更全（替代开源，构造商业生态圈）
配置更好（零代码配置、可视化配置）
组件化程度高（在通用技术上封了一层）
