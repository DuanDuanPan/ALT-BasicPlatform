## 1. 目标

---

## 2. 规范规则

1. 蓝图正文与附录只使用相对路径，不再使用作者本地绝对路径。
2. 每条证据必须区分：`已采纳`、`部分采纳`、`已阅读未采纳`。
3. 若仓库存在 `git commit hash`，优先记录 commit；若不存在，则记录文件版本日期、最后修改时间和内容摘要。
4. 证据链必须指向“本次版本真正使用的输入”，不是把所有读过的资料平铺进去。

---

## 3. 当前工作区版本依据

当前工作区不是 `git` 仓库，无法提供 commit hash。

因此本次采用以下替代版本依据：

- 文件名自带版本/日期
- 最后修改时间
- 文件大小
- `SHA-256` 摘要

该组合足以支持非作者复核输入版本。

## 6. 采纳口径说明

### 6.1 已采纳

满足以下任一条件即标记为“已采纳”：

- 直接支撑 `v0.5` 主文结论
- 直接支撑 `Must/Should/Deferred` 附录内容
- 被用于 ADR 章节映射或实施切片定义

### 6.2 部分采纳

满足以下情况：

- 只吸收其中的部分结构或术语
- 原始材料内容较多，但本版只采纳一部分方向

### 6.3 已阅读未采纳

满足以下情况：

- 作为背景阅读存在
- 未形成本版明确结论
- 或被更高优先级来源覆盖

## 7. 对主文的约束

从 `v0.5` 开始，主文 `§14` 的引用必须遵守：

- 不再写本地绝对路径
- 不再混合“已采纳”和“仅阅读”
- 若无 git commit，则至少提供时间 + 摘要
