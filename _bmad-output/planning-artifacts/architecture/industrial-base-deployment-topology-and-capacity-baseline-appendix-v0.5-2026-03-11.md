# 工业研发底座部署拓扑与容量基线附录

状态：`Draft v0.5`  
日期：`2026-03-11`  
对应修订项：`S-04`

---

## 1. 目标

本附录给出八个部署区之间的网络边界、最小资源基线和扩展方向参考。

---

## 2. 网络拓扑原则

1. 北南向流量统一经入口区
2. 管理流量与业务流量隔离
3. 集成区和标准交换区不得直通数据区
4. 边缘区只通过受控通道进入中心

---

## 3. 部署区与流量方向

| 区域 | 允许进入 | 允许访问 | 禁止直连 |
| --- | --- | --- | --- |
| 入口区 | 外部用户、浏览器、SSO | 应用区、网关 | 数据区 |
| 工程客户端区 | Workbench、插件、工具代理 | 集成区、应用区 | 数据区、管理区 |
| 应用区 | L2/L3 服务 | 平台区 | 数据区直连 |
| 平台区 | L4 服务 | 数据区、集成区 | 外部直连 |
| 数据区 | 平台区 | 存储内部互联 | 外部、工程客户端 |
| 集成区 | 连接器、API Gateway、文件交换 | 平台区、标准交换区 | 数据区直连 |
| 标准交换区 | TODS/ATF/XML、跨网代理 | 集成区、平台区 | 应用区直连 |
| 边缘区 | 站点代理、缓存、协议适配 | 集成区、标准交换区 | 数据区、管理区 |

---

## 4. 最小生产基线

| 区域 | 最小基线 | 扩展方向 |
| --- | --- | --- |
| 入口区 | 双实例网关/WAF | 水平扩容 |
| 应用区 | 关键 L2/L3 服务双实例 | 水平扩容 |
| 平台区 | Identity、ACL、Audit、Trace、Config&Release 双实例 | 水平扩容 |
| 数据区-关系库 | 主备或集群 | 垂直优先，按域拆分 |
| 数据区-搜索 | `3` 节点起步 | 水平扩容 |
| 数据区-图谱 | `3` 节点起步 | 水平扩容 |
| 数据区-对象存储 | 多副本或纠删码 | 水平扩容 |
| 边缘区 | 单站点双进程或双节点 | 站点级横向扩展 |

---

## 5. 租户隔离对非关系存储的要求

| 存储 | 逻辑隔离模式要求 |
| --- | --- |
| 搜索引擎 | 租户索引前缀或独立索引空间 |
| 图数据库 | 租户子图/命名图或独立图实例 |
| 对象存储 | 租户桶前缀或独立桶 |
| 时序库 | 租户标签 + 保留策略隔离 |
| 消息系统 | 租户主题前缀或独立 topic namespace |

---

## 6. 跨网摆渡原则

- 通过标准交换区或文件交换通道进行
- 必须具备审批引用、内容校验和审计记录
- 不允许通过临时脚本或人工拷贝作为标准方案

---

## 7. 容量估算维度

正式容量规划至少应给出以下变量：

- 日活用户数
- 峰值并发 API 请求
- 日新增 `DataSet` 批次
- 单站点日采集量
- 日均 `TraceLink` 增量
- 日均导出包数量
- HPC 日均作业数和结果包大小

---

## 8. 高可用建议

- 网关、入口、平台共享服务：双实例起步
- 审计、发布、Trace：故障不得静默降级
- 边缘节点：允许单站点故障，但不得拖累中心交易链

---

## 9. 运维红线

- 任何环境不得把对象存储、搜索、图谱和交易库共盘部署
- 管理区账户不得复用业务登录会话
- 逻辑隔离模式下，不得只做关系库隔离而忽略搜索/图谱/对象存储隔离
