## 执行摘要

经过三轮多角色深度讨论，核心结论如下：

1. **蓝图方向正确**——"产品族共用平台内核"的策略能够实现历史产品线整合与降本增效目标
2. **降本增效可量化**——统一共享服务年省 ~150 人月（~450 万），投资回收期约 8 个月
3. **快速复制是降本增效的最大杠杆**——通过行业模板包（ITP），新客户交付可从 5-6 个月压缩到 3-4 周（降幅 80%）
4. **蓝图需要补强三个关键机制**——行业模板包、元模型扩展服务提升至 P0、四层扩展治理体系

## 第二轮：产品线整合与降本增效分析

### 重复建设消除分析

| 能力           | CDM        | TDM        | MIS         | 重复度   | 蓝图解法            | 降本 |
| -------------- | ---------- | ---------- | ----------- | -------- | ------------------- | ---- |
| 用户/组织/权限 | 自建       | 自建       | 自建        | 🔴 三套   | Identity Service    | ⬇️ 高 |
| 主数据         | 未独立     | 内置字典   | 独立系统    | 🔴 三套   | Master Data Service | ⬇️ 高 |
| 文件/对象管理  | MinIO+版本 | 大文件存储 | Tomcat 附件 | 🟡 两套半 | File/Object Service | ⬇️ 中 |
| 流程/审批      | 工作流引擎 | 简化版     | 审批流      | 🔴 三套   | Workflow Service    | ⬇️ 高 |
| 搜索           | ES 全文    | 数据检索   | 无          | 🟡 一套半 | Search Service      | ⬇️ 中 |
| 图谱/谱系      | neo4j      | 无         | 无          | 🟢 一套   | Trace Service       | ⬇️ 低 |

> **结论：** 身份权限 + 主数据 + 流程引擎统一，可消除 6-9 个重复维护模块，3 年节省 54-81 人月。

## 第三轮：领域模型快速复制与平台扩展机制

### 行业模板包（ITP）架构

新增架构概念 `Industry Template Package`：

```
Industry Template Package (ITP):
├── object-models/           # 领域对象模型定义（继承核心+行业专属）
├── workflows/               # 流程模板（审批流+任务流）
├── views/                   # 页面/表单/看板模板
├── rules/                   # 规则配置
├── connectors/              # 预置连接器配置
├── master-data/             # 预置主数据（字典、编码、分类）
├── permissions/             # 权限模板
└── manifest.yaml            # 元数据+版本+依赖声明
```

### 快速复制效果对比

| 阶段     | 当前模式     | 底座模板包模式       |
| -------- | ------------ | -------------------- |

### 关键工程决策

**SchemaVersion 需支持 3-way merge：**

```yaml
SchemaVersion:
  baseVersion: string          # 底座基线版本
  parentVersion: string        # 模板版本
  localDelta:                  # 客户差量
    addedFields: []
    modifiedFields: []
    removedFields: []          # 标记不物理删除
  mergeStrategy: auto|manual|locked
```

**存储实现策略：**

| 对象类型                            | 存储方案             |
| ----------------------------------- | -------------------- |
| 核心对象（Requirement, Project...） | 多表继承 + 扩展表    |
| 柔性领域对象（TestSpec 派生等）     | JSON 扩展 + 文档库   |
| 所有对象统一注册到 MetaObjectType   | 运行时按注册信息路由 |
