# 工业研发底座非功能架构基线附录

状态：`Draft v0.5`  
日期：`2026-03-11`  
对应修订项：`D-02`

---

## 1. 目标

本附录补齐 `v0.5` 最小非功能架构视图，覆盖性能、弹性容错、数据生命周期、合规映射和成本基线。

---

## 2. 性能模型视图

## 2.1 关键路径延迟预算

| 路径 | 预算 |
| --- | --- |
| 中心读请求 | `P95 < 800ms` |
| 中心写请求 | `P95 < 1.5s` |
| 导出审批前置判定 | `P95 < 1s` |
| `TraceLink` 投影收敛 | 正常情况下 `< 30s` |
| 边缘批次元数据同步 | `< 5min` |
| 模板包初始化 | `< 1 天` |

## 2.2 吞吐热点

- 网关读写流量
- `TraceIntent` 到 `TraceLink` 投影链路
- `DataSet` 导入链
- 边缘批次同步链
- HPC 回调和结果包登记链

---

## 3. 弹性与容错视图

## 3.1 实施层位

| 策略 | 实施层 |
| --- | --- |
| 限流 | 入口区、API Gateway |
| 熔断 | 集成区、外部依赖调用边界 |
| 重试 | Outbox 投影器、边缘同步、HPC 回调 |
| 降级 | 边缘离线、本地 ACL 缓存、只读视图 |
| 死信 | 事件总线、边缘同步链、Trace 投影链 |

## 3.2 降级原则

- 审计失败时，高危动作不得降级放行
- 密钥服务失败时，不得降级为明文模式
- 图谱/搜索投影失败时，可不阻断主事务，但必须阻断基线冻结和正式发布

---

## 4. 数据生命周期视图

| 阶段 | 典型对象 | 策略 |
| --- | --- | --- |
| 创建 | Requirement、Project、Test、DataSet | 主事务入主存，建立审计与主线意图 |
| 激活/发布 | Deliverable、SchemaVersion、ITP | 进入发布治理和版本账本 |
| 热存 | 近期 `DataSet`、`ResultPackage` | 快速访问、支持检索和分析 |
| 温存 | 阶段性结果与评审资料 | 支持复盘与只读映射 |
| 冷存 | 长期归档结果、旧版本报告 | 对象存储归档、WORM 或只读策略 |
| 销毁/失效 | 临时导出包、过期缓存、废弃配置 | 按审批与审计规则失效或销毁 |

补充规则：

- 审计记录不允许物理删除
- `TraceLink` 关系优先失效标记，不做静默删除
- 数据销毁必须保留销毁审批和回执

## 5. 合规映射视图

| 合规关注点 | 对应技术能力 |
| --- | --- |
| 身份鉴别 | `Identity Service`、MFA、会话控制 |
| 最小权限 | `ACL Policy Service`、范围授权、动作授权 |
| 涉密控制 | `Classification & Secrecy Policy Service`、导出/跨网审批 |
| 审计留存 | `Audit Service`、WORM/归档、审计查看权限 |
| 加密与密钥 | `Crypto Policy Service`、密钥治理、传输加密 |
| 三员分立 | 管理区隔离、互斥角色、前置拦截与审计 |
| 隔离部署 | 单实例/逻辑隔离/物理隔离三态 |
| 跨网交换 | 标准交换区、文件交换、审批与审计 |

说明：

- 本表提供架构映射基线，不替代正式等保/分保/军工保密条款清单
- 后续若进入正式合规评审，应按具体标准逐条映射

## 6. 成本模型视图

## 6.1 成本构成

- 入口与网关成本
- 平台共享服务计算成本
- 数据区存储与集群成本
- 边缘节点与站点运维成本
- HPC 和对象存储容量成本
- 运维、审计和治理运营成本

## 6.2 三种部署模式的相对成本

| 模式 | 成本等级 | 说明 |
| --- | --- | --- |
| 单实例模式 | 低 | 一套服务和一套数据区，适合单法人单单位 |
| 逻辑隔离模式 | 中 | 共享基础设施但需多租户隔离、索引隔离和治理复杂度 |
| 物理隔离模式 | 高 | 独立集群、独立存储、独立搜索/图谱/消息，适合高保密场景 |

## 6.3 成本优化原则

- 共享服务优先复用，不按产品线重复部署
- 热温冷分层降低高价值存储占比
- 边缘节点按站点能力分级，不做一刀切高配
- 物理隔离模式只用于确有合规要求的场景

---

## 7. 运维与治理成本红线

- 为满足逻辑隔离而忽略搜索/图谱/对象存储隔离，属于伪节省
- 依赖人工运维脚本支撑升级回滚，长期成本会高于前期治理投入
- HPC 结果与业务元数据不分离，会显著抬高交易存储与备份成本

---

## 8. 当前结论

`v0.5` 的非功能视图已经具备最小实施基线：

- 有性能目标
- 有弹性与容错层位
- 有数据生命周期规则
- 有合规映射基线
- 有三种部署模式的相对成本判断

后续版本需要继续补足更细的容量数字和正式合规逐条映射，但“完全缺失非功能视图”的问题已消除。
