# 工业研发底座安全运行时序与错误码附录

状态：`Draft v0.5`  
日期：`2026-03-11`  
对应修订项：`M-06`

---

## 1. 安全执行总顺序

所有请求统一遵循以下顺序：

1. 网络来源和接入区校验
2. 身份鉴别与会话校验
3. 租户/法人/项目隔离校验
4. 角色、对象、范围授权
5. 密级与保密规则判定
6. 解密/导出/跨网审批凭据校验
7. 业务执行
8. 审计落地与告警

任何一步失败，后续步骤不再执行，但审计和告警仍需执行。

---

## 2. 读请求时序

1. `API Gateway` 校验来源网络、TLS 和基础令牌
2. `Identity Service` 校验会话、SSO 和 MFA 状态
3. `ACL Policy Service` 校验租户、角色、对象范围和动作权限
4. `Classification & Secrecy Policy Service` 校验对象密级与用户密级差
5. 需要明文读取敏感数据时，`Crypto Policy Service` 校验解密授权
6. 业务服务执行查询
7. `Audit Service` 写入访问审计

---

## 3. 写请求时序

1. 网关校验接入来源与会话
2. `Identity` 校验用户主体
3. `ACL` 校验动作权限和对象范围
4. `Classification` 校验目标对象可修改密级范围
5. 业务服务执行命令并提交主事务
6. `Audit` 写入对象变更、状态变化和关键字段修改记录
7. 如果触发导出、发布、跨网或加解密动作，再进入对应二级审批链

---

## 4. 导出请求时序

1. 通过 `Identity` 和 `ACL` 完成基础访问校验
2. `Classification` 判断对象密级、导出范围和脱密要求
3. 若需要审批，则校验审批单、时效、一次性令牌和审批人角色
4. `Crypto` 确认导出内容的加解密策略
5. 业务服务生成导出任务
6. `Audit` 记录导出请求、审批引用、导出包标识和下载行为

---

## 5. 跨网/边缘同步时序

1. `Integration Gateway` 校验接口、站点、证书和网络边界
2. `Identity` 或边缘本地凭据缓存校验主体身份
3. `ACL` 校验站点允许上传/下载的数据范围
4. `Classification` 校验密级和跨网策略
5. `Crypto` 校验传输加密、密钥和解密授权
6. 边缘同步代理执行传输
7. `Audit` 记录批次、方向、审批引用和站点标识

---

## 6. 三员分立实现规则

### 6.1 角色约束

- 系统管理员：负责运行配置，不得查看审计明文和审批自身高危变更
- 安全管理员：负责安全策略和密级规则，不得执行普通业务发布
- 审计管理员：负责查看和抽查审计，不得修改安全策略和系统配置

### 6.2 API 层实现

- 前置 `Interceptor/Gateway Filter`：识别管理面、高危动作和管理角色
- 策略层 `ACL/Policy Engine`：校验互斥角色与越权访问
- 后置 `Audit Hook`：无论成功或失败都写入审计

## 7. 错误码规范

| 错误码 | HTTP | 含义 | 是否必须审计 |
| --- | --- | --- | --- |
| `SEC-AUTH-401` | `401` | 未认证、令牌失效或会话过期 | 是 |
| `SEC-AUTH-403` | `403` | 已认证但无动作权限 | 是 |
| `SEC-TENANT-403` | `403` | 租户、法人或项目隔离不匹配 | 是 |
| `SEC-CLASS-403` | `403` | 密级或保密范围不满足 | 是 |
| `SEC-DECRYPT-403` | `403` | 解密或明文读取未获授权 | 是 |
| `SEC-EXPORT-409` | `409` | 导出审批不存在、过期或范围不匹配 | 是 |
| `SEC-CROSSNET-409` | `409` | 跨网/边缘同步策略不允许 | 是 |
| `SEC-MUTEX-409` | `409` | 三员分立互斥规则冲突 | 是 |
| `SEC-AUDIT-500` | `500` | 审计写入失败 | 是，且应触发告警 |
| `SEC-KEY-500` | `500` | 密钥服务调用失败 | 是，且应触发告警 |

---

## 8. 审计最小字段

每条安全审计记录至少包含：

- `requestId`
- `actorId`
- `tenantId`
- `action`
- `objectType`
- `objectId`
- `securityLevel`
- `decision`
- `decisionReason`
- `approvalRef`
- `sourceIp / sourceSite`
- `occurredAt`

---

## 9. 失败处理

- `SEC-AUDIT-500`：应视为高危事件，阻断高危写、导出和跨网动作
- `SEC-KEY-500`：禁止回退到明文模式
- 边缘离线场景：仅允许使用短时缓存的审批凭据和最小 ACL 策略，不允许生成永久权限
